KNOWLEDGE FLOW

원문과 해석을 분리하는 위키

LLM에게 맡기는 일과 사람이 검증해야 할 일을 파일 구조로 나눕니다.

01

raw

원천 자료, 캡처, 링크, PDF를 보존합니다. 요약 오류가 생겨도 되돌아갈 기준점입니다.

02

wiki

자료별 요약, 개념 문서, 종합 문서를 분리해 LLM이 읽기 쉬운 지식 구조로 바꿉니다.

03

tools

위키 링크와 필수 파일을 점검합니다. 현재는 `wiki_lint.py`로 기본 검증을 수행합니다.

04

log

중요한 판단, 변경 이력, 다음 작업을 남겨 다음 세션에서도 맥락을 이어갑니다.

LOCAL AI WORKBENCH

웹 공개용과 로컬 실행용을 분리합니다

Vercel은 공개 가이드와 문서 탐색에 적합하고, Ollama 모델 실행은 로컬 Mac에서 처리하는 편이 안전합니다. Local AI Workbench는 LLMwiki, MCP 논문, GitHub 도구 조사 문맥을 선택적으로 모델에 전달합니다.

HARNESS ENGINEERING

좋은 프롬프트보다 중요한 작업 환경

하네스 엔지니어링은 장시간 실행되는 AI 에이전트가 읽고, 판단하고, 검증하고, 이어받을 수 있는 환경을 설계하는 일입니다. LLMwiki에서는 `AGENTS.md`, `wiki/log.md`, `tools/wiki_lint.py`, 원천과 해석의 분리가 그 역할을 맡습니다.

MULTI-AGENT CASE STUDY

TradingAgents로 보는 의사결정 하네스

TradingAgents는 트레이딩 회사를 흉내 낸 LLM 멀티에이전트 연구 프레임워크입니다. LLMwiki에서는 투자 도구가 아니라 역할 분리, 토론, 리스크 검토, 최종 승인 구조를 연구하는 사례로 다룹니다.

Analysts

자료 수집과 해석

펀더멘털, 뉴스, 시장 심리, 기술지표를 역할별로 나누어 분석합니다.

Debate

상승/하락 관점 토론

한쪽 결론으로 빨리 수렴하지 않고 기회와 위험을 분리해 검토합니다.

Risk

리스크 점검

최종 결정 전에 변동성, 노출, 실패 가능성을 별도 단계로 확인합니다.

GUIDEBOOK

현재 저장소에 포함된 문서

상세 가이드북 사용법, 효과, 프로그램 위치, 운영 루틴 하네스 엔지니어링 Codex와 장시간 에이전트를 위한 작업 환경 설계 Local AI Workbench Ollama와 위키 문맥을 연결하는 로컬 앱 유용한 GitHub 저장소 법률 검색, 음성 에이전트, 화면 녹화, 3D 생성 도구 TradingAgents 금융 멀티에이전트 의사결정 구조와 하네스 사례
