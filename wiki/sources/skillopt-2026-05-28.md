# SkillOpt 공개와 논문 v2 팩트체크 2026-05-28

## 자료 개요

SkillOpt는 Microsoft가 공개한 LLM 에이전트 skill 자동 최적화 프레임워크다. 모델 가중치를 학습하지 않고, 에이전트가 참고하는 자연어 skill 문서를 학습 대상으로 삼는다. 공식 프로젝트 페이지는 SkillOpt를 frozen language agent의 compact natural-language skill document를 trainable state로 다루는 text-space optimization 방식으로 설명한다.

연관 자료는 세 가지다.

| 자료 | 용도 |
| --- | --- |
| 프로젝트 페이지 | 방법 요약, 결과 표, 데모와 citation 제공 |
| GitHub 저장소 | 설치 방법, 코드, 라이선스, `best_skill.md` 산출물 구조 확인 |
| arXiv v2 논문 | 최신 초록, train/validation/test split, 성능 수치 인용 기준 |

## 핵심 주장

SkillOpt의 핵심 주장은 "프롬프트나 skill 문서도 검증 가능한 학습 대상이 될 수 있다"는 것이다. 실행 궤적을 수집하고, 별도의 optimizer model이 성공·실패 사례를 반영해 add/delete/replace 방식의 제한된 수정을 제안하며, held-out validation 성능이 향상될 때만 후보 skill을 채택한다.

이 구조는 단순한 self-editing과 다르다.

| 요소 | SkillOpt에서의 의미 |
| --- | --- |
| Rollout | 현재 skill로 에이전트를 실행해 messages, tool calls, verifier feedback, final score 등을 수집 |
| Reflection | 성공·실패 minibatch를 나누어 반복 오류와 보존할 행동을 분석 |
| Bounded edit | skill 문서 전체 재작성 대신 add/delete/replace 단위의 작은 수정 적용 |
| Validation gate | train 결과가 아니라 held-out selection/validation 성능으로 후보 채택 |
| Rejected-edit buffer | 실패한 수정 방향을 기억해 같은 오류 반복을 줄임 |
| Export | 최종 산출물을 `best_skill.md` 같은 단일 skill artifact로 배포 |

## 팩트체크 메모

사용자 요약의 큰 방향은 맞지만, LLMwiki에는 다음처럼 정정해 기록한다.

1. "테스트셋 검증 통과 시 업데이트"가 아니라 "held-out selection/validation split 통과 시 업데이트"로 표현한다. test split은 최종 성능 보고에 사용된다.
2. 성능 수치는 프로젝트 페이지와 arXiv v2가 다르다. 프로젝트 페이지 표는 GPT-5.5 기준 Codex +21.8, Claude Code +18.6을 제시한다. arXiv v2 초록은 direct chat +23.5, Codex +24.8, Claude Code +19.1을 제시한다. 위키 본문에서는 논문 v2 수치를 우선 인용하고, 불일치 메모를 남긴다.
3. "regression 방지"는 완전 방지로 쓰지 않고, bounded edit, validation gate, rejected-edit buffer를 통한 "성능 붕괴 위험 완화 설계"로 표현한다.

## LLMwiki 적용 가치

SkillOpt는 LLMwiki의 하네스 엔지니어링, 프롬프트 라이브러리, Codex·Claude Code·Cline류 코딩 에이전트 운영과 직접 연결된다. 특히 `AGENTS.md`, 작업 규칙, 테스트 실행 절차, 문서 작성 템플릿을 사람이 수동 개선하는 단계에서 더 나아가, 실행 로그 기반으로 작은 수정 후보를 만들고 검증 통과 시만 반영하는 SkillOps 패턴으로 확장할 수 있다.

LLMwiki에서는 SkillOpt를 다음 주제와 연결해 다룬다.

- [[Harness Engineering|하네스 엔지니어링]]: 에이전트 실행 환경과 검증 루프
