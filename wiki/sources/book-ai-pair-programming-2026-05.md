## 핵심 자산 1 — 4S 원칙

좋은 프롬프트를 짜기 위한 4가지 원칙. 책의 2.3절에서 제시된 약어.

| 약어 | 의미 |
| --- | --- |
| **S**ingle | 하나의 질문 혹은 작업에 집중하라 |
| **S**pecific | 명확하고 상세하게 지시하라 |
| **S**hort | 간결하게 핵심을 전달하라 |
| **S**urround | 주변 컨텍스트를 제공해서 질문의 상황을 잘 전달하라 |

### 4S 보조 패턴

| 패턴 | 효과 |
| --- | --- |
| 답변 형식 지정(JSON/XML/시작 어구 등) | 후속 처리 자동화, 형식 강제 |
| 구역 태그(`<메뉴>...</메뉴>` 같은 범위 한정) | 할루시네이션 방지 |
| 반복("다시", "더 짧게", "더 구체적으로") | 결과물 점진 개선(티키타카) |

## 핵심 자산 2 — 7가지 프롬프팅 테크닉

책의 2.4절에서 정리한 기법.

| 테크닉 | 핵심 |
| --- | --- |
| **zero-shot prompting** | 별도 정보·컨텍스트 없이 지시 (간단한 번역·지식 설명에 적합) |
| **one-shot prompting** | 기대 결과 예시 1개 제공 (출력 형식 고정에 효과적) |
| **few-shot prompting** | 예시 2개 이상으로 결과 일관성 향상 (단, 복잡한 추론은 한계) |
| **Chain of Thought (CoT)** | 추론 과정의 예시를 보여줘 단계적 사고 유도 |
| **zero-shot CoT** | "한 단계씩 생각해봐" 한 줄로 단계적 사고 유도 |
| **Tree of Thought (ToT)** | "여러 전문가가 각자 풀고 결론을 비교하라" — 다관점 분석 |
| **ReAct** | Reason + Act 반복(추론 → 검색·실행 → 다시 추론) — 에이전트형 |

## 책 본문 활용 시 주의

- 책 본문의 예시 코드·대화 캡처·다이어그램은 골든래빗(주) 저작권 보호 대상이다. 위키에는 **개념 명칭·원칙 구조만 짧게 인용**하고 본문 직접 복사는 하지 않는다.
- GitHub 예제 코드(`jiyeonseo/book-ai-pair-programming`)는 별도 라이선스를 확인한 후 참조한다.

## 한계와 갱신 계획

- **도구 버전 노후화**: 2024년 봄 기준(GPT-4, Claude 3, Copilot v1.178)이라 2026년 5월 시점에서 도구 UI·기능 설명은 일부 폐기. 단 4S 원칙과 7개 테크닉은 도구 무관.
- **갱신 시점**: 2026년 11월(약 6개월 후) GPT-5/Claude 4 환경에서 본 카드의 원칙·테크닉이 여전히 유효한지 재평가. 새 도구 도입 시(예: Antigravity CLI 정착) 부록 A 활용도 재검토.
- **신간 비교**: 같은 저자 또는 동일 주제의 후속 도서가 출간되면 본 카드를 fork 형식으로 갱신(`book-ai-pair-programming-2026-XX.md`).
