# AI 답변 물타기 방지 커스텀 인스트럭션 메모 2026-05-28

## 자료 개요

사용자 제공 메모를 바탕으로 정리한 커스텀 인스트럭션 패턴이다. ChatGPT, Claude, Gemini, Grok 같은 LLM이 지나치게 친절하거나 과포장된 답변, 질문 재포장, 중립적 양비론, 과도한 목록화를 반복하는 문제를 줄이기 위한 응답 스타일 규칙이다.

외부 원문 URL이 없는 자료이므로, LLMwiki에서는 특정 커뮤니티 유행이나 공식 프롬프트로 단정하지 않고 "사용자 제공 운영 패턴"으로 기록한다.

## 핵심 주장

이 패턴의 핵심은 AI를 "공감 리액션 중심"이 아니라 "정보 전달과 판단 중심"으로 튜닝하는 것이다. 좋은 답변은 친절한 포장보다 다음 요소를 우선해야 한다.

| 원칙 | 의도 |
| --- | --- |
| 불필요한 리액션 제거 | 빈말 없이 바로 본론으로 들어가기 |
| 과도한 포맷팅 제한 | 매번 헤더·목록을 남발하지 않기 |
| 정직 모드 | 문제를 돌려 말하지 않고 먼저 지적하기 |
| 가정 공개 | 답변 전제, 해석, 트레이드오프를 명시하기 |
| 반복 제거 | 질문 재진술과 불필요한 요약 줄이기 |
| 확신 수준 표시 | 사실, 추론, 추측을 구분하기 |
| 의견 요청 시 하나 선택 | A/B 질문에서 책임 있는 판단을 내리기 |

## 검토 메모

반영 가치가 높다. 특히 LLMwiki의 문서 작성, 코드 리뷰, 연구 메모, 정책 비교, 도구 추천에서 답변 밀도와 신뢰도를 높이는 규칙으로 쓸 수 있다.

다만 주의할 점도 있다.

- "리액션 금지"를 너무 강하게 걸면 상담, 교육, 협업 상황에서 차갑게 느껴질 수 있다.
- "리스트 사용 금지"는 분석 문서에는 오히려 가독성을 낮출 수 있으므로 "필요할 때만 사용"이 적절하다.
- "하나 선택" 규칙은 의사결정에는 좋지만, 고위험 영역에서는 불확실성과 조건을 함께 표시해야 한다.
- "모르면 모른다"는 규칙은 반드시 출처 확인, 웹 검색 필요성, 추론 여부 표시와 함께 써야 한다.
