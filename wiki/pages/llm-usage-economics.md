# LLM 사용 경제학

## 요약

LLM 산업이 SaaS형 구독 모델에서 사용량 기반(usage-based) 인프라 모델로 전환되는 흐름을 정리한 문서다. 시점 고정 사실은 출처 카드에 분리하고, 본 페이지는 정책 변경과 무관하게 유지되는 영속적 원칙(작업 분류 모델, 로컬 LLM 전략, 운영 가이드)에 집중한다.

## 거시 명제

> **3대 프런티어 랩이 동시에 인터랙티브 사용과 프로그래매틱 사용의 회계 기준을 분리하고 있다. 이것은 가격 인상이 아니라 LLM 산업의 회계 모델이 SaaS에서 인프라 기반으로 옮겨가는 구조 전환이다.**

이 명제는 다음 두 가지 사실에 기반한다.

1. 에이전트 1회 실행은 채팅 1회보다 수십~수백 배 토큰을 소모한다(파일 읽기 → 분석 → 명령 실행 → 오류 재시도 → 테스트 루프).
2. 공급자는 이 사용 패턴을 더 이상 정액 구독으로 흡수하기 어렵다.

## 사용자 유형별 영향 구분

| 사용자 유형 | 영향 |
| --- | --- |
| 일반 채팅 사용자 | **영향 없음** — 구독 그대로 유지 |
| 자동화·CLI 반복 호출 사용자 | **영향 큼** — 별도 크레딧·API·쿼터로 분리 |
| 대량 에이전트 운용 사용자 | **영향 매우 큼** — 로컬 LLM 또는 엔터프라이즈 경로 전환 검토 |

이 구분은 정책이 바뀌어도 골격이 유지된다. 어떤 공급자든 이 세 유형 중 어디에 속하는지 먼저 판정하면 비용 노출도가 결정된다.

## 작업 분류 3계층(Tier) 모델

영구 적용 가능한 분류 원칙이다. 시점·공급자 가격 변동과 무관하게 작업을 어디에 배치할지 결정하는 기준이다.

### Tier 1 — 클라우드 프런티어 모델

| 적합 작업 | 이유 |
| --- | --- |
| 논문 구조 설계 / 투고용 문장 품질 개선 | 추론·문체 품질이 로컬 대비 우위 |
| 복잡한 코드 설계 판단 | 도메인 폭넓은 추론 |
| 오류 원인 추론 | 다중 가설 비교 능력 |
| 최종 검토 / 중요 제안서 논리 정리 | 책임 영역의 안전 마진 |

### Tier 2 — 무료/저비용 인터랙티브 도구

| 도구 | 용도 | 비용 |
| --- | --- | --- |
| NotebookLM | 자료 요약·정리, 출처 기반 질의 | 무료 |
| Google AI Studio | 프롬프트 설계·테스트 | 무료 |
| 클라우드 채팅 무료 티어 | 일회성 질의 | 무료 |

### Tier 3 — 로컬 LLM

| 적합 작업 | 적합도 |
| --- | --- |
| 강의자료/원고 초안 생성 | 높음 |
| 코드 주석 / README 정리 | 높음 |
| 문서 요약 / 표 형식 변환 | 높음 |
| 반복 프롬프트 테스트 | 높음 |
| 간단한 Python/JS 코드 생성 | 중간~높음 |
| 고난도 아키텍처 설계 | 중간 |
| 복잡한 학술 심사 대응 | 낮음~중간 |

## 작업 유형별 도구 선택 매트릭스

| 작업 유형 | 권장 처리 방식 |
| --- | --- |
| 학술 최종 문장, R&R 답변, 투고 전략 | Tier 1 (프런티어 모델) |
| 강의안 초안, 슬라이드 대본 초안, 퀴즈 생성 | Tier 3 (로컬 LLM) |
| 코드 대량 리팩터링 초안 | Tier 3 → Tier 1 최종 검토 |
| MCP 서버 반복 테스트 | Tier 3 또는 API mock |
| 자동화 에이전트 장시간 실행 | Tier 3 또는 저가 API |
| 고난도 디버깅 | Tier 1 (단기 집중) |
| 문서 분류·요약·태깅 | Tier 3 또는 소형 API |

## 로컬 LLM 인프라 구성

### 단계별 권장 구성

| 단계 | 도구 | 목적 |
| --- | --- | --- |
| 1단계 | Ollama + Open WebUI | 빠른 로컬 챗봇, 문서 초안, 코드 보조 |
| 2단계 | llama.cpp의 `llama-server` | OpenAI 호환 API로 자체 도구·MCP 연결 |
| 3단계 | vLLM | 본격 서버화 (병렬 처리·OpenAI 호환 API 서버) |

### 플랫폼별 지원

| 도구 | Windows 네이티브 | 권장 도입 순서 |
| --- | --- | --- |
| Ollama | 지원 | 1순위 (즉시 도입) |
| llama.cpp (llama-server) | 지원 | 2순위 (OpenAI 호환 API 필요 시) |
| LM Studio | GUI 지원 | Ollama 보완재로 병행 |
| vLLM | WSL2 필요 | 본격 서버화 단계 |

### 하드웨어 가이드 (12GB VRAM 기준)

| 모델 | 양자화 | 용도 |
| --- | --- | --- |
| Qwen 2.5 Coder 14B | Q4 | 코드 주석, 리팩터링 초안 |
| Gemma 2 9B | Q5 | 문서 요약, 초안 생성 |

## 운영 원칙

### 라이브 시연 3중 백업 구조

| 백업 계층 | 역할 |
| --- | --- |
| 1차: 클라우드 프런티어 모델 | 메인 시연 |
| 2차: 로컬 LLM | 네트워크·쿼터 장애 시 즉시 전환 |
| 3차: 사전 녹화 영상/스크린샷 | 두 계층 모두 실패 시 학습 흐름 보존 |

### 학습 환경 등급 모델

| 등급 | 구성 | 대상 |
| --- | --- | --- |
| Bronze | 무료 클라우드 도구 + 무료 로컬 모델 | 입문 학습자 |
| Silver | 저가 구독 1종 + 로컬 모델 | 중급 학습자 |
| Gold | 프런티어 모델 구독 + 로컬 서버 + API 키 | 고급/연구 학습자 |

### 위험 시나리오 대응

| 시나리오 | 1차 대응 | 2차 대응 |
| --- | --- | --- |
| 구독 한도 소진 | Tier 3 로컬 LLM으로 전환 | 작업 우선순위 재조정 |
| 공급자 정책 갑작스러운 변경 | 영향 범위 즉시 진단 | 자매 도구·로컬 모델 병행 |
| 라이브 시연 중 네트워크 장애 | 로컬 LLM 즉시 활성화 | 사전 녹화 자료 재생 |
| 자동화 스크립트 비용 폭발 | Extra usage 옵션 OFF로 강제 정지 | 사용처 전수 진단 후 재구성 |

## 재사용 가능한 진단 도구

### `-p` 사용처 점검 명령어

```bash
# macOS / Linux
grep -r "claude -p" .
grep -r "gemini -p" .
grep -rE "codex (exec|-p)" .

# Windows PowerShell
Get-ChildItem -Recurse | Select-String "claude -p"
Get-ChildItem -Recurse | Select-String "gemini -p"
```

### Extra usage 옵션 결정 가이드

| 선택 | 동작 | 적합 사용자 |
| --- | --- | --- |
| OFF (기본) | 한도 초과 시 자동화 정지 | 비용 예측 가능성 우선 |
| ON | 한도 초과분을 API 가격으로 자동 과금 | 가용성 우선, 비용 모니터링 체계 보유 |

### 일일 소진율 계산식

```
일일 안전 소진율 = 월간 크레딧 ÷ 30일
                 = $20 / 30
                 ≈ $0.67/일 (Pro 기준)
```

3일·7일·14일 단위 누적치를 추적하면 월말 초과 여부를 사전 감지할 수 있다.

## 교육 적용 시 고려사항

- 학생에게 가르치는 LLM 활용 비용 모델이 정책 전환과 함께 바뀐다. 캡스톤·졸업 작품에서 에이전트를 쓰는 경우 ROI 계산이 통째로 달라진다.
- 평가 정책에 "AI 사용 기록 제출 필수화"를 도입하면, 학생의 비용 인식과 학습 흐름을 동시에 추적할 수 있다.
- 학과 단위로 로컬 LLM 공유 서버를 운영하면, 학생 1인당 비용 노출 없이 Tier 3 작업을 흡수할 수 있다.

## 다음 작업

1. 공급자 정책 추가 변경 시 출처 카드를 새로 만들고, 본 페이지의 원칙이 여전히 유효한지 검토한다.
2. 로컬 LLM 운용 결과 데이터를 축적해 Tier 3 적합 작업 목록을 실증 기반으로 업데이트한다.
3. 학기 종료 후 작업 분류 매트릭스의 실제 사용 비율(Tier 1 vs Tier 3)을 측정해 ROI 섹션을 추가한다.
