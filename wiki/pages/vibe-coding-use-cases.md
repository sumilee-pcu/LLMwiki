# 바이브 코딩 활용 사례

## 요약

연구 중심 AI 융합 교육자 정체성과 병렬로 진행 중인 프로젝트(논문, 강의, 연수, 게임공학 수업, 자격증 학습 등)를 고려해 바이브 코딩 활용 사례 10가지를 적합도 기준으로 분류한 문서다. easy-vibe 레포 평가([[easy-vibe-2026-05-20]])와 자매 문서로, 외부 자료의 사실 정보가 아닌 개인 워크플로우 설계 관점에서 정리했다.

## 높은 적합도 (즉시 적용 권장)

| --- | --- | --- |
| 4 | 프로젝트별 4단 폴더 구조 (MD+컨텍스트 / 프로토타입 / MVP / 아카이브) | 후속 연구, Tetris 게임, 교사 연수, 네트워크관리사 학습 등 병렬 프로젝트를 동일 구조로 표준화 |

## 핵심 시사점

### 1. #4 + #8 결합이 가장 큰 레버리지

폴더 구조(#4)와 차시 PDF 세트화(#8)의 결합이 가장 큰 레버리지를 만든다. 한 학기 강의를 "지침 MD + 차시별 세트메뉴(지도안·pptx·학습지)"로 표준화하면, AI 융합 큐레이터로서의 작업 흐름이 동료 교수와 연수 수강자에게도 그대로 전수 가능한 교수 자산이 된다.

### 2. #10 바이브코딩 학습용 웹앱이 기존 강점과 가장 정렬

이미 GitHub → Vercel 흐름을 5월 21일 워크숍 포스터에서 다루고 있고, "설명하기 어려운 개념의 동적 시각화"는 정보(학부) → 지능(대학원)이라는 교육철학과 직접 연결된다.

### 3. #1 + #3 통합으로 학기 단위 운영 대시보드

학사일정·연구일정·연수일정을 단일 뷰에서 추적하는 형태는, 현재 병렬로 돌아가는 다수 프로젝트의 인지 부하를 크게 줄여준다.

2. 5월 21일 워크숍 자료에 #10 사례를 데모로 포함할지 검토한다.

4. 학기 종료 후 적합도 평가를 갱신하고, 실제 적용 결과를 기반으로 본 문서의 별점·우선순위를 재조정한다.
